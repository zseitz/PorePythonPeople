# doc-sync-rules skill

Purpose: keep repository surfaces consistent whenever behavior/contracts change.

## Required synchronization

- Update `Docs/components.md` when component behavior/contracts change.
- Update `Docs/nanoporethon_textbook.md` when user workflow/operational guidance changes.
- Append a concise row to `Docs/agent_logs/REQUEST_LOG.md`.

## Traceability heuristics

- Keep request-log notes factual and short.
- List touched files and objective clearly.
- Do not claim verification outcomes that were not executed.

## Anti-hallucination checklist

- Surface-consistent: docs match actual code behavior.
- Traceable: request-log row present.
